Members

Member nameDescription
Free The Free mode simply transposes all [Constants._WS_](s) between each [Term] of a query to a boolean representation ().


e.g

C#
string query = "fornax is awesome"
.
Free interpretes the query in [fornax OR is OR awesome] which inturn works a retrieval of documents with either ("fornax","is","awesome")
NOTE: Query Analysis is required but not shown in this example.
Native The Native mode simply transposes all [Constants._WS_](s) between each [Term] of a query to a boolean representation ().


e.g

C#
string query = "fornax is awesome"
.
Native interpretes the query in [fornax AND is AND awesome] which inturn works a retrieval of only documents with all ("fornax","is","awesome").
NOTE: Query Analysis is required but not shown in this example.
Natural This Represents a natural Language query mode. Natural tells fornax to process a query as a natural language query.

e.g

C#
string naturalQuery = "give me all my comics""
. Natural interpretes the query into ["give"[verb] , "all"[quantity-all] , "comics"(noun-query)]...This format is then returned for further analysis by fornax, afterwards fornax inturn works a retrieval of documents with ("comic": in all [FieldScope]s and see[Zone]) as well as doucuments with FornaxFormat as comicbook.
NOTE: Only Use this option if and only if a natural language exclusive search is needed i.e. Natural never process a query as Free or Native do.
Normal The Normal mode simply taste's a query before tagging for interpretation.

e.g

C#
string query = "fornax is awesome"
. Normal tastes the query and there by interpretes the query in [fornax OR is OR awesome] which inturn works a retrieval of documents with either ("fornax","is","awesome")
NOTE: Query Analysis is required but not shown in this example.
